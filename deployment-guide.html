Deploy to RHEL Server

Step-by-step guide for deploying QA Infinity on a Red Hat Enterprise Linux server

Docker Compose RHEL 8 / 9 4 Services Non-developer friendly

Before You Start — What You Need

  • SSH access to the RHEL server (IP address + username + password)
  • Server must have internet access to download Docker images
  • The qa-infinity project folder on your Windows computer
  • WinSCP installed on your Windows computer (free download — see Step 4)
  • Production API keys (OpenRouter, SMTP email credentials)
1

Connect to the Server

Open a terminal on your computer and connect to the RHEL server using SSH. You will be asked for your password — the cursor won't move while you type, that is normal.

Terminal 
# Replace with your actual username and server IP address
ssh your-username@<SERVER-IP-ADDRESS>

# Example:
ssh albin@192.168.10.50
2

Install Docker (one-time setup)

Run these commands to install Docker on the server. You only need to do this once. Each line installs a different part of Docker.

Shell — Install Docker 
# Update the system's package list
sudo dnf update -y

# Add Docker's official repository for RHEL
sudo dnf install -y dnf-plugins-core
sudo dnf config-manager --add-repo https://download.docker.com/linux/rhel/docker-ce.repo

# Install Docker
sudo dnf install -y docker-ce docker-ce-cli containerd.io docker-compose-plugin

# Start Docker and make it auto-start on server reboot
sudo systemctl start docker
sudo systemctl enable docker

# Allow your user to run Docker without "sudo" every time
sudo usermod -aG docker $USER

# Apply the group change immediately
newgrp docker

Verify Docker installed correctly — you should see version numbers printed:

Verify 
docker --version
docker compose version
3

Open the Firewall Ports

The app runs on two ports. The server's built-in firewall must be told to allow traffic through them, otherwise browsers on the network won't be able to reach the app.

Shell — Firewall Rules 
# Port 3000 = the web UI (what users open in their browser)
sudo firewall-cmd --permanent --add-port=3000/tcp

# Port 4000 = the backend API
sudo firewall-cmd --permanent --add-port=4000/tcp

# Apply the changes
sudo firewall-cmd --reload
4

Copy the Project Files to the Server

Since we are not using Git, copy the project folder directly from your Windows computer to the server. Choose one of the two methods below — Option A (WinSCP) is recommended for non-developers as it uses a visual interface.

💡 WinSCP is a free drag-and-drop file transfer app for Windows. It works like Windows Explorer but connects to Linux servers.

4a. Download and install WinSCP

Go to winscp.net and download the free installer. Run it and accept the defaults.

4b. Connect to the server

  1. Open WinSCP.
  2. In the login window, set File protocol to SFTP.
  3. Fill in Host name → your server IP (e.g. 192.168.10.50)
  4. Fill in User name and Password for the server.
  5. Click Login. Accept the server fingerprint warning if it appears.

4c. Navigate to the home folder on the server

The right panel shows the server. Navigate to /home/your-username/ (WinSCP usually opens here automatically).

4d. First — prepare the project folder on Windows

Before copying, delete these two folders from the project on your Windows machine (they are very large and not needed on the server — the server will regenerate them):

  • node_modules (inside the root and inside each package folder)
  • local_artifacts
  • .git (hidden folder — enable "Show hidden files" in Windows Explorer to see it)
⚠️ Do NOT delete these from your original working copy. Make a copy of the folder first, then delete from the copy before uploading.

4e. Drag and drop to upload

  1. On the left panel (your Windows machine), navigate to the qa-infinity project folder.
  2. Select the entire qa-infinity folder.
  3. Drag it to the right panel (the server) and drop it.
  4. WinSCP will show a progress bar. Wait for it to complete.
✅ When done, the qa-infinity folder will appear in the right panel under /home/your-username/.
💡 SCP uses the same SSH connection as Step 1 but copies files. Run these commands in PowerShell on your Windows computer (not on the server).

4a. Open PowerShell on your Windows machine

Press Win + X → click Terminal or PowerShell. Make sure you are NOT already connected to the server via SSH for this step.

4b. Copy the project to the server

Windows PowerShell 
# Replace the path, username, and server IP with your actual values
scp -r "C:\Users\albin\Sixdee telecom solutions pvt. ltd\AirtelAfrica-Ventas - Documents\Delivery\Automation\QA Infinity\qa-infinity" your-username@<SERVER-IP>:~/qa-infinity

# Example:
scp -r "C:\...\qa-infinity" albin@192.168.10.50:~/qa-infinity
⚠️ The node_modules folders are very large and will slow the transfer significantly. If possible, delete them from a copy of the folder before running this command, then let the server install them fresh in Step 6.

4f. (Both methods) — Verify the files arrived on the server

Go back to your SSH session (Step 1) and confirm the folder is there:

Server — SSH Terminal 
# List the home directory — you should see qa-infinity listed
ls ~

# Enter the project folder
cd ~/qa-infinity

# Confirm the files are there
ls

You should see files like docker-compose.yml, package.json, and the packages/ folder.

5

Create the Environment File

The .env file was copied along with the project in Step 4. It currently contains development values. You must update it with the correct production settings before starting the app.

Shell 
# Make sure you are inside the project folder
cd ~/qa-infinity

# Open the .env file in the text editor
nano .env

A text editor will open. Update these values — use the arrow keys to navigate, type to edit:

Setting What to change it to
NODE_ENV=development Change to NODE_ENV=production
CORS_ORIGIN=http://localhost:3000 Change to http://<SERVER-IP>:3000
APP_URL=http://localhost:3000 Change to http://<SERVER-IP>:3000
JWT_SECRET=... Replace with a long random string (see below)
OPENROUTER_API_KEY=... Paste the real production OpenRouter API key
SMTP_USER= Add the email address for sending reports
SMTP_PASS= Add the email app password

To generate a strong JWT_SECRET, run this in a separate terminal and paste the output:

Generate Secret 
openssl rand -hex 64
💾 Save and exit nano: press Ctrl + X, then Y, then Enter.
6

Build and Start the App

This single command builds all 4 services and starts them in the background. The first run takes 5–15 minutes — Docker is downloading base images and compiling code. A lot of text will scroll by; that is normal.

Shell 
docker compose up --build -d
ℹ️ --build compiles the app from source (required on first run and after any code update). -d runs everything in the background so the app keeps running after you close the terminal.
7

Check that all 4 containers are healthy:

Shell

You should see all four services with a green Up or running status:

⚠️ If any container shows Exited or Error, check its logs with: docker compose logs qa-api
8

Access the App

Open a browser on any computer on the same network and navigate to:

Browser URL 
http://<SERVER-IP-ADDRESS>:3000

# Example:
http://192.168.10.50:3000
✅ You should see the QA Infinity login page. Deployment is complete!

Day-to-Day Operations

Stop the app

docker compose down

Start the app

docker compose up -d

Update after a code change

# Re-upload files via WinSCP, then:
docker compose up --build -d

Watch live logs

docker compose logs -f qa-api

Check disk usage

docker system df

Free up disk space

docker system prune -f
ℹ️ To stop watching live logs, press Ctrl + C.

Troubleshooting

❌ "Permission denied" when running Docker commands
Run newgrp docker in the terminal, or log out of the server and log back in.
❌ Browser shows "This site can't be reached"
Confirm firewall ports are open (Step 3). Double-check the server IP address is correct. Make sure you ran docker compose up -d.
❌ qa-api container keeps restarting
Run docker compose logs qa-api to read the error. Most commonly a missing or misspelled value in the .env file.
❌ App loads but AI features don't work
Verify OPENROUTER_API_KEY is correctly set in the .env file and that the key has not expired or run out of credits.
❌ "No space left on device" error
Run docker system prune -f to remove unused images and containers and free up disk space.
❌ Email reports are not being sent
Check that SMTP_USER and SMTP_PASS are filled in the .env file. For Gmail, use an App Password, not your regular account password.
🔒
Security reminder: The .env file contains API keys and secrets. Never share it, email it, or commit it to Git. Treat it the same as a password. If you suspect a key has been leaked, rotate it immediately from the provider's dashboard.
