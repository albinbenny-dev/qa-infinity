QA Speed
Requirement → Test in Minutes

Drop a Jira story, PDF, or URL. The Writer Agent extracts acceptance criteria, generates structured test cases with use-case tags, and queues script generation — in one action. No human translation required between requirement and test.

Resilience
Tests That Fix Themselves

When a test fails due to a UI change, the Healing Agent captures the DOM, identifies selector drift or flow changes, and proposes a patch with a confidence score. Above the threshold, it auto-applies and re-runs. QA engineers review proposals rather than debug scripts.

Zero Handoff
Fully Automated End-to-End

The Orchestration Agent wires all stages into a single automated pipeline. A Jira story moving to "Ready for QA" can trigger TC generation, script writing, test execution, healing, and email report — with no human in the loop unless confidence thresholds require it.

NEW — Core Platform Feature

Orchestration
Agent

The Orchestration Agent is the command layer that connects every other agent into a single automated pipeline. One external trigger — a Jira story, a CI/CD webhook, a scheduled cron, or an API call — fires the full QA lifecycle from TC generation through published report, with configurable human checkpoints at each stage.

Pipeline Configuration Options
  • Stage toggles: Each stage can be enabled/disabled per pipeline definition. Run execution-only or generation-only pipelines.
  • Human checkpoint gates: Any stage can be configured to pause and send a notification until a QA engineer approves before continuing.
  • Auto-approve threshold: TC generation and healing patches above the confidence threshold proceed without human review.
  • Scope filtering: Pipeline can be scoped to a specific use-case group, suite, or tag — e.g. "run only Checkout Flow tests after payment service deploy".
  • Environment routing: Different pipeline definitions can target different env configs (UAT vs Staging vs Prod).
  • Retry policy: Max heal attempts, retry delays, and fallback to human review configurable per pipeline.
  • Parallel pipelines: Multiple pipelines can run simultaneously for different scopes. Concurrency controlled per project.

Agent Intelligence Layer

The 8-Agent
Intelligence Pipeline

Each agent is a specialized LLM-backed microservice. They share context via a common ProjectContext store and communicate via BullMQ job queues and Socket.io events.

AGENT 00 NEW
Orchestration Agent
The pipeline command centre. Listens for external triggers (Jira webhooks, CI/CD calls, cron), sequences all other agents in the correct order, manages human approval gates, handles retries and failures, and routes results to configured output destinations. Makes the platform fully autonomous end-to-end.
Triggers
Jira webhook CI/CD POST API Cron schedule Manual / REST Chat command
Produces
Pipeline run record Stage status events Published report Jira / email / Slack output
Key Capabilities
  • Sequential or parallel stage execution based on dependency graph
  • Per-stage human approval gates with configurable timeout + fallback
  • Confidence-based auto-approve for TC generation and healing patches
  • Scope filtering — run only affected use-case groups from CI payload
  • Real-time pipeline status board in the UI (stage-by-stage progress)
  • Retry with exponential backoff on transient failures
  • Multi-pipeline concurrency — independent pipelines per project
AGENT 01
Test Case Writer Agent
Ingests requirements from multiple simultaneous sources and synthesises them into structured, deduplicated test cases with AI-assigned use-case tags. Respects project requirement library for background context.
Accepts
Jira stories (batch) PDFs / Word / Excel URLs Seed test cases Req library (auto) Agent learnings
Produces
Structured test cases UseCase tags (AI-assigned) Dedup report
  • Multi-input queue: Jira + docs + URLs processed simultaneously in one session
  • UseCase consistency engine: reuses existing labels before creating new ones
  • Seed TC style mirroring: matches depth, format and coverage of reference TCs
  • ProjectContext enrichment: uses agent learnings and scan context
AGENT 02 NEW
UI Scanner Agent
Crawls the AUT using a real Playwright browser, maps every navigation node, extracts semantic locators per page, validates the login flow, and stores everything in ProjectContext. Acts as the project's institutional memory of the application under test.
Accepts
Base URL + credentials Scan depth config
Produces
Navigation tree Page locator map Login instructions UseCase summary
  • Saves login steps, selectors, and post-login URL to ProjectContext
  • Extracts semantic locators (role, label, testid) per page and stores per URL pattern
  • Context auto-loaded by Writer + Script agents for richer generation
  • Re-scan on demand when AUT structure changes
AGENT 03 NEW
Agentic Browser Trace
Claude drives a real Playwright browser through a target menu via tool-use loop (screenshot → tool call → observe), recording every verified selector and action. Returns test cases and saves learnings to ProjectContext. Generates scripts from verified action log without extra LLM calls.
Accepts
Target URL + menu context Seed steps (optional) Additional context
Produces
Verified test cases Action log (for script gen) Agent learnings saved
  • 8 browser tools: click, fill, navigate, assert_visible, scroll, wait_for, finish
  • Stateless turns (no compounding screenshots) — cost ~$0.08–0.12 per trace
  • 20-step max · scoped to target URL only, no app-wide crawl
  • Verified locators saved → future generations use real selectors automatically
AGENT 04
Script Writer Agent
Converts approved test cases into production-ready Playwright TypeScript scripts with smart locator strategy, POM classes, and API request builders. Followed immediately by automatic script verification.
Accepts
Approved test cases ProjectContext (locators) Agent learnings Custom .spec.ts files
Produces
Playwright TS scripts Page Object Models API test files Script verification status
  • Locator priority: role → label → testid → placeholder → css fallback
  • Script verification pipeline: QUEUED → GENERATING → VERIFYING → VERIFIED / MANUAL_REVIEW
  • Golden script promotion: reference scripts for healing and future generation
  • Prerequisite TC support: links setup (login/nav) script as a reusable dependency
AGENT 05
Execution Agent
Orchestrates test runs with surgical precision. Supports instant individual runs, multi-select batch runs, use-case group runs, and cron-scheduled suites. Real-time log streaming via Socket.io.
Run Modes
Individual ▶ button per TC UseCase group run Multi-select batch Suite run Cron schedule
  • Live log streaming per test via Socket.io — visible in Execution page
  • Screenshot + trace + video capture per run result
  • Trigger types: MANUAL / SCHEDULED / INDIVIDUAL / GROUP / HEAL_RERUN
  • Multi-environment: select env config (UAT / Staging / Prod) per run
AGENT 06
Self-Healing Agent
Post-failure intelligence. Captures DOM snapshot and error trace, identifies root cause (selector drift, flow change, API schema change), and generates a confidence-scored patch. Auto-applies above threshold; proposes below.
Heal Types
SELECTOR drift FLOW change API_SCHEMA drift
Patch States
PENDING AUTO_APPLIED APPROVED REJECTED EXHAUSTED
  • Diff viewer: before/after code comparison with line-level highlighting
  • Confidence scoring (0–1.0); auto-apply threshold configurable per project
  • Golden script reference: healing preserves patterns from promoted golden scripts
AGENT 07
Reports & Dashboard Agent
Transforms raw execution data into actionable intelligence. Real-time dashboards, AI failure analysis, trend analytics, and configurable email dispatch — all within the platform.
Produces
Live dashboard AI failure analysis Run trend chart Flaky test heatmap Email report (HTML) Excel export
  • Per-run: pass/fail/skip counts, duration, AI root-cause summary
  • Playwright trace viewer embedded for failed test case inspection
  • Video playback per run result inline in the Reports page
  • Configurable email recipients with trigger events (on failure / always)
AGENT 08 NEW
Chat Agent
Conversational AI assistant with persistent memory and cross-agent action routing. Accessible from any page via the floating sidebar. Routes natural language commands to the correct underlying agent and returns results in-chat.
Example Commands
"Run all checkout tests" "Why did TC-0078 fail?" "Generate TCs for VEN-4821" "Schedule nightly smoke at 2am"
  • Multi-turn conversation with persistent ChatMemory per project
  • Action cards: structured results returned inline (test cases, run results, heal proposals)
  • File attachment support: drop a PDF or screenshot into chat for analysis
  • Routes to: Writer, Execution, Reports, Healing, or Orchestration agents based on intent

Platform Capabilities

Complete Feature
Reference

F-01 NEW
Orchestration Pipelines
End-to-end automation from trigger to report. Configurable per-stage human gates, confidence-based auto-approve, scope filtering by use-case or tag, multi-environment routing, real-time pipeline status board in the UI.
F-02 NEW
UI Scanner + ProjectContext
Automated AUT crawl that builds a living knowledge base: navigation map, per-page semantic locators, validated login flow, use-case summary. All data stored in ProjectContext and auto-injected into TC and script generation. Re-scan on demand.
F-03 NEW
Agentic Browser Trace
Claude-driven Playwright loop that generates test cases from a real browser session. Verified selectors and flows saved as Agent Learnings — future generations for the same menu use real locators automatically. Script generation skips LLM for traced TCs.
F-04 NEW
Agent Learnings
Verified locators, flows, and selectors from browser traces are persisted in ProjectContext.agentLearnings. Every subsequent generation — TC writing, script writing — is enriched with real, verified UI knowledge. Learnings accumulate per project over time.
F-05
TC Library — UseCase Grouped
Full test case catalog with collapsible use-case groups, coverage metrics, multi-select, drag-to-reassign, bulk move/delete/approve, inline edit, filter by type/status/tag, and one-click "Send to Execution" for any group or selection.
F-06
Multi-Type Testing
UI tests via Playwright browser automation. API tests via request builder. SIT tests via multi-service chained scenarios. All three types generated by the Writer Agent and executed by the same runner with type-aware assertions.
F-07 NEW
Script Verification Pipeline
Generated scripts pass through an automatic verification step. Scripts that pass are marked VERIFIED; those with quality concerns are flagged MANUAL_REVIEW. Golden scripts can be promoted as reference baselines for healing and future generation.
F-08
Self-Healing Agent
Detects selector drift, flow changes, and API schema changes post-failure. Confidence-scored patches: above threshold = auto-applied; below = proposed for engineer review. Before/after diff viewer. Healing attempts tracked and bounded per job.
F-09 NEW
Chat QA Agent
Floating chat sidebar with persistent project memory. Natural language commands routed to the correct agent. Returns structured action cards inline. Supports file attachment (PDF, screenshot) for analysis. Covers TC generation, execution trigger, run analysis, and scheduling.
F-10
Multi-Project Workspaces
Fully isolated projects with independent test libraries, scripts, runs, schedules, and dashboards. URL-routed: /projects/:slug/. Global dashboard for admin-level cross-project health view. Copy test cases and scripts between projects.
F-11
Reports & AI Analysis
Per-run: pass/fail/skip counts, AI root-cause summary, flaky test heatmap, run trend chart. Playwright trace viewer embedded. Video playback per failed TC. Email HTML report with configurable recipients and trigger events.
F-12
Scheduler
Built-in cron engine (node-cron) — no external dependency. Visual schedule builder with cron expression editor. Schedules survive container restarts. Per-schedule email recipients. Supports test case sub-sets, full suites, or use-case groups.
F-13
Requirement Library
Per-project requirement document store. Upload HLD, BRD, PDF, Word, or Excel files via the UI. Auto-loaded as background context on every Writer Agent run. New documents picked up on next run. Reduces repeated uploads to zero.
F-14 NEW
LLM Usage Dashboard
Every LLM call logged with token counts, cost, agent name, and project. Usage dashboard shows per-agent token consumption over time. Enables cost governance across teams and projects. Admin-level platform-wide view available.
F-15
Custom Script Upload
Drop existing .spec.ts or .spec.js files directly into the script library. Indexed and tagged as first-class citizens alongside AI-generated scripts. Can be promoted to golden status. Full execution engine compatibility.
F-16
Zero-Dependency Client
Entire QA infrastructure runs inside Docker. Engineers need only a browser. No Node.js, no Playwright, no Python on the accessing machine. Works from Windows, Mac, or Linux. Team access via local network or VPN.
F-17 NEW
Prerequisite TC Linking
A test case can declare a prerequisite whose script handles the login and navigation setup steps. Script generation uses the prerequisite's verified action log, avoiding redundant login sequences in every script. Managed via the Edit TC modal.
F-18 NEW
Environment Config Management
Per-project named environment profiles (UAT, Staging, Production, custom) with URL, username, and password. One profile can be set as default. Env configs used by UI Scanner, Agentic Trace, and all run types. Credentials stored encrypted.
